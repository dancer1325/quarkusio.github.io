* _Example:_ how to configure the default OTLP gRPC Exporter

[source,properties]
----
# == OpenTelemetry `Resource` attribute /
#   appears | ALL telemetry data
#   == application | telemetry is created
#   by default, artifactId
quarkus.application.name=myservice

# == gRPC endpoint | send the telemetry
#   by default, http://localhost:4317
quarkus.otel.exporter.otlp.endpoint=http://localhost:4317

# == gRPC headers
#   OPTIONAL
quarkus.otel.exporter.otlp.headers=authorization=Bearer my_secret

# | log messages, add tracing information
quarkus.log.console.format=%d{HH:mm:ss} %-5p traceId=%X{traceId}, parentId=%X{parentId}, spanId=%X{spanId}, sampled=%X{sampled} [%c{2.}] (%t) %s%e%n

# | console log (!= access log)
quarkus.http.access-log.pattern="...traceId=%{X,traceId} spanId=%{X,spanId}"
----


* connection related properties
    ** signal agnostic configurations (== SAME properties | tracing & metrics)
+
[source,properties]
----
quarkus.otel.exporter.otlp.endpoint=http://localhost:4317
----

    ** different configurations / EACH signal
+
[source,properties]
----
# traces exporter endpoint
quarkus.otel.exporter.otlp.traces.endpoint=http://trace-uri:4317

# metrics exporter endpoint
quarkus.otel.exporter.otlp.metrics.endpoint=http://metrics-uri:4317

# logs exporter endpoint
quarkus.otel.exporter.otlp.logs.endpoint=http://logs-uri:4317
----
